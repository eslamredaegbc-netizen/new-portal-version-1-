# منصة الرصد الإعلامي لقطاع الإعلام

الجهة المالكة: قطاع التطوير التقني  
الحقوق: جميع الحقوق محفوظة لقطاع التطوير التقني

منصة رصد حديثة بواجهة `JavaScript` مع backend منظم بـ `FastAPI` وقاعدة بيانات `SQLite` داخلية، مخصصة للبحث متعدد المصادر، تحليل النتائج، اختيار المصادر يدويًا، ثم إصدار تقارير رصد فعلية.

## التشغيل

1. `pip install -r requirements.txt`
2. `uvicorn app:app --host 0.0.0.0 --port 8000`

ثم افتح:

- `http://127.0.0.1:8000`

## بيانات الدخول الافتراضية

- اسم المستخدم: `admin`
- كلمة المرور: `Admin@123`

## ماذا يفعل النظام

- بحث متعدد المصادر عبر:
  - الأخبار
  - المواقع الرسمية
  - Facebook
  - Instagram
  - YouTube
  - المنتديات
  - الويب العام
  - الصور
  - الروابط المباشرة
- دعم `Google Dorking` داخل خطة البحث.
- إدخال `سبب البحث` ليبني النظام خطة بحث مفسرة قبل التنفيذ.
- دمج النتائج المتشابهة في `Cases`.
- تصنيف النتائج إلى:
  - شكوى
  - استغاثة
  - طلب مساعدة
  - انتقاد
  - تشهير
  - إشادة
  - خبر محايد
  - غير ذي صلة
  - مكرر
- تلوين النتائج:
  - أحمر: الخطورة الأعلى
  - أصفر: المحايد أو الأقل حساسية
  - أخضر: الإشادة أو النتائج الإيجابية
- تحليل مصادر مجمل داخل النظام.
- اختيار يدوي للمصادر التي تدخل التقرير النهائي.
- إصدار تقارير فعلية بصيغ:
  - `PDF`
  - `DOCX`
  - `CSV`
  - `JSON`
- حفظ التقارير ومصادرها داخل قاعدة البيانات.
- مساعد داخلي بعنوان: `اسأل الذكاء الاصطناعي` يجيب فقط من بيانات النظام مع أدلة وثقة.

## الهيكل

- `app.py`: نقطة التشغيل وتطبيق `FastAPI`
- `static/`: الواجهة الأمامية `HTML / CSS / JavaScript`
- `monitoring_app/storage.py`: قاعدة البيانات الداخلية
- `monitoring_app/services/`: خدمات البحث، التخطيط، التحليل، التقارير، والمساعد
- `data/`: قاعدة البيانات المحلية
- `exports/`: ملفات التقارير المخرجة

## Google Web Search

يدعم النظام الآن البحث عبر `Google Web Search` داخل تدفق البحث، وليس `Google News` فقط.

لتفعيل Google داخل النظام اضبط متغيرات البيئة التالية:

- `GOOGLE_SEARCH_API_KEY`
- `GOOGLE_SEARCH_ENGINE_ID`

إذا لم تُضبط هذه المتغيرات، يستمر النظام بالعمل تلقائيًا عبر مزودات البحث العامة الأخرى دون توقف.

تظهر شفافية مزود البحث داخل تفاصيل النتيجة نفسها في خانة سبب ظهور النتيجة.

## ملاحظة مهمة حول Google

يعتمد التكامل على خدمة Google الرسمية:

- `Programmable Search Engine`
- `Custom Search JSON API`

وبحسب توثيق Google الرسمي الحالي:

- كل طلب يحتاج `API key` و`Engine ID` و`query`
- قدرات `Search the entire web` تمر بمرحلة انتقال في بعض سيناريوهات Google الحديثة، لذلك قد تعتمد الإتاحة الفعلية على أهلية الحساب أو إعداد المحرك المستخدم

## ملاحظات تشغيلية

- لا يمكن تقنيًا ضمان دقة `100%` أو `صفر أخطاء` على الويب المفتوح.
- يرفع النظام الدقة عمليًا عبر:
  - تعدد المصادر
  - خطة بحث مفسرة
  - استعلامات متعددة
  - تحليل سياقي وتحفظي
  - مراجعة بشرية نهائية للمصادر قبل التقرير
- رصد `Facebook` و`Instagram` يعتمد على المحتوى العام المفهرس والمتاح علنًا، وليس المحتوى الخاص أو المغلق.
- `OCR` واستخراج نص الفيديو يعملان عند توفر النصوص أو الوسائط العامة وعند توفر المكتبات المناسبة.
