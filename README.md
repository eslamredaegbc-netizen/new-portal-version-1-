# منصة الرصد الإعلامي لقطاع الإعلام

الجهة المالكة: قطاع التطوير التقني  
الحقوق: جميع الحقوق محفوظة لقطاع التطوير التقني

منصة رصد إعلامي متعددة المصادر، مبنية لتعمل مباشرة على `Streamlit` مع قاعدة بيانات داخلية `SQLite` ومحرك بحث وتحليل وتقارير داخل نفس النظام.

## التشغيل

1. `pip install -r requirements.txt`
2. `streamlit run app.py`

## تشغيل Streamlit Cloud

- اجعل ملف التشغيل الرئيسي هو: `app.py`
- لا تستخدم `api_app.py` كملف رئيسي على Streamlit Cloud، لأنه خاص بالنسخة البرمجية `FastAPI`

## تشغيل اختياري للنسخة البرمجية API

إذا أردت تشغيل واجهة الـAPI والنسخة `JavaScript` القديمة محليًا:

- `uvicorn api_app:app --host 0.0.0.0 --port 8000`

## بيانات الدخول الافتراضية

- اسم المستخدم: `admin`
- كلمة المرور: `Admin@123`

## ما الذي يقدمه النظام

- بحث متعدد المصادر عبر:
  - الأخبار
  - المواقع الرسمية
  - Google Web Search عند ضبط مفاتيحه
  - Facebook
  - Instagram
  - YouTube
  - المنتديات
  - الويب العام
  - الصور
  - الروابط المباشرة
- دعم `Google Dorking`
- إدخال سبب البحث لبناء خطة بحث مفسرة
- تصنيف النتائج إلى:
  - شكوى
  - استغاثة
  - طلب مساعدة
  - انتقاد
  - تشهير
  - إشادة
  - خبر محايد
  - غير ذي صلة
  - مكرر
- تلوين النتائج:
  - أحمر للحالات الأعلى خطورة
  - أصفر للمحتوى المحايد أو الأقل حساسية
  - أخضر للمحتوى الإيجابي
- دمج النتائج المتشابهة في قضايا `Cases`
- تقارير فعلية بصيغ:
  - `PDF`
  - `DOCX`
  - `CSV`
  - `JSON`
- حفظ التقارير والمصادر المختارة داخل قاعدة البيانات
- مساعد داخلي بعنوان: `اسأل الذكاء الاصطناعي`

## Google Web Search

لتفعيل Google داخل تدفق البحث اضبط متغيرات البيئة التالية:

- `GOOGLE_SEARCH_API_KEY`
- `GOOGLE_SEARCH_ENGINE_ID`

إذا لم تضبطها، سيستمر النظام بالعمل عبر مزودي البحث الآخرين دون توقف.

## ملاحظات تشغيلية

- `app.py` هو الآن ملف Streamlit الأساسي الجاهز للنشر على Streamlit Cloud
- `api_app.py` يحتفظ بنسخة `FastAPI` السابقة حتى لا نفقد تكامل الـAPI
- لا يمكن تقنيًا ضمان دقة `100%` أو صفر أخطاء على الويب المفتوح، لكن النظام يرفع الدقة عمليًا عبر تعدد المصادر وخطة البحث المفسرة والتحليل التحفظي
